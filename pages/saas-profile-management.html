Case Study

SaaS Profile Management QA Sample

Manual QA documentation sample covering test planning, test cases, bug reports, validation behavior, failed saves, cancel behavior, and persistence after refresh.

Feature Under Test

User Profile Management for a SaaS web application. The feature allows a user to view and update profile information such as first name, last name, phone number, and job title.

Testing Objective

Verify that users can update profile information successfully and that the system handles validation, cancel behavior, failed save scenarios, and saved data persistence correctly.

Scope

  • Profile page access
  • Profile information updates
  • Required field validation
  • Phone number validation
  • Save and cancel behavior
  • Failed save error messaging
  • Persistence after refresh

Sample Test Coverage

Test Area Example Scenario Expected Result
Page access User opens the profile page Current profile information displays
Valid update User edits first and last name, then saves Confirmation message appears and changes persist
Required field User clears first name and saves Validation message appears and form is not saved
Cancel behavior User edits fields, then clicks Cancel Unsaved changes are discarded
Backend failure Save request fails Clear error message appears and data is not incorrectly saved

Defects Documented

  • Save button remains enabled after a successful update
  • Invalid phone number is accepted without validation
  • Cancel button saves edited values unintentionally

AI-Assisted QA Workflow

For a workflow like this, I may use AI to draft a first test matrix, check whether common negative cases are missing, rewrite test cases for clarity, and summarize open risks. I still decide what to test, validate the behavior, investigate defects, and determine release risk.
